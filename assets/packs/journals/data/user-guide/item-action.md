# What Is an Action? {#action-overview}

An Action is an executable procedure attached to an actor or nested within
another item. Actions represent things a character can actively do — making
a skill test, performing a special maneuver, activating a unique ability,
or triggering any other defined procedure. Actions provide buttons on the
character sheet that players and GMs can click to initiate the procedure,
streamlining gameplay by packaging complex rule sequences into a single
click.

# Where It Appears {#action-where}

Actions can belong to any actor type (Beings, Cohorts, Structures, Vehicles,
Assemblies) and can also be nested inside other items. For example, a weapon
might contain an Action for a special attack, or a skill might include an
Action for its standard test. Actions appear on the actor sheet's **Actions**
tab as well as inline within the items they are nested under.

Actions are the most flexible item type in SoHL — they serve as the
system's primary mechanism for attaching custom executable behavior to
any document.

# Key Properties {#action-properties}

- **Name** — what the action is called (displayed on the button).
- **Procedure** — what happens when the action is triggered (the logic
  that executes).
- **Description** — an explanation of what the action does and when to
  use it.
- **Parent Item** — if nested, which item this action belongs to.
