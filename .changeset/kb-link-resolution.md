**KB: resolve `{@link}` references in the developer docs (#435)**

The knowledgebase developer docs now cross-link to the API site instead of
rendering literal `{@link ...}` text. The content-prep step
(`utils/build-kb-content.mjs`) resolves inline `{@link}` / `{@linkcode}` /
`{@linkplain}` tags against the TypeDoc symbol map (`kb/data/api-symbols.json`):

- `sohl.*` targets — including `Class.member` forms, which carry the member anchor
  — become Markdown links to the symbol's page on api.heroiclands.org; `@linkcode`
  renders the text as code.
- Optional display text (`{@link Target | text}` / `{@link Target text}`) is
  honored; otherwise the last dotted segment is the link text.
- External-URL targets become plain links.
- References the map doesn't know (the `{@link Symbol.member}` syntax examples)
  degrade to a code span — never a broken link, never a build failure.

`kb/data/api-symbols.json` is now a committed, generated artifact (regenerated by
`npm run docs`, like `type-catalog.md`) so the KB deploy — which does not run
TypeDoc — has it. This is the Hugo re-implementation of the resolution the retired
Astro KB carried.
