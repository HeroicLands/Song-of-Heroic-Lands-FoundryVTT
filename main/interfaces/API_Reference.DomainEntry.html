Interface DomainEntry

A single entry in the world-scoped Domain registry.

Domains represent thematic groupings such as schools of magic, deities, totems, birthsigns, or natural lore traditions. They are reference data, not Items — they live in a Foundry world setting and are accessed exclusively through DomainRegistry.

Shortcodes are dotted, three-segment strings of the form <source>.<group>.<name> (e.g. sohl.hexhodai.pyrethos, sohl.totem.Bear, sohl.asguardian.thorr). The first segment matches the source field. System defaults preserve human-readable casing in the third segment.

interface DomainEntry {
    ancestorShortcode?: string;
    description: string;
    family: DomainFamily;
    iconFAClass: string;
    img: string;
    label: string;
    shortcode: string;
    sort: number;
    source: string;
}

OptionalancestorShortcode

ancestorShortcode?: string

Optional shortcode of another DomainEntry this one derives from. Used to express hierarchical relationships such as a religion pointing at the deity it worships, or a sub-school of magic pointing at its parent school. Resolution is by string match against the registry; the parent does not need to exist (and dangling parents are intentionally allowed for the non-existent-deity case).
