Remarks

Declared as a precise object literal (no widening) so per-kind constructor types survive — that is what ItemLogicByKind derives from. The runtime registry COMMON_ITEM_LOGIC is intentionally widened (see below) to preserve string-keyed indexing by consumers such as SohlDataModel.create.
